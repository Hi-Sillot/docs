<Aside>stable rc beta alpha S RC B A 是版本号保留字段，不应在渠道中使用</Aside>

渠道制将是汐洛宝典的默认发行方式，渠道将决定用户可以使用的特性，以及特性更新的周期。docker hub 率先支持， github 将在后续支持（部分渠道工作流将转移至下游）。

<Aside type="tip" title="发行说明">

渠道是发行版本 0.35 引入的概念
编写发行说明的渠道有：新彖乄渠道、洛可可渠道、先锋渠道。
版本类型：发行版本、内部版本、基线版本、代号版本
内部版本、基线版本、代号版本均为发行版本 0.35 引入

每个代号版本都有一个专门的演示视频

</Aside>

<Aside type="tip" title="命名规范">

1. 不得包含 `&`
2. 使用下划线连接

</Aside>

## 洛可可渠道

> 洛可可渠道是海文东标准的一部分

洛可可渠道包括已冻结的特性和已稳定的未冻结特性，由先锋渠道禁用实验性特性后传递，调试能力弱，适合大多数用户。

  <Card title="Rococo" icon="seti:plan">

    在 docker hub ，洛可可渠道使用 `Rococo` 和 `R` 标签，额外使用 `latest` 这一标签。

    更新周期较长（2月左右）。

    - 发行名称：汐洛·洛可可
    - 发行版本号更新 `0.100.*` -> `0.101.*` -> `0.102.*` ，重大修复版本可例外
    - 内部版本号更新 `1.36.*.{{ yyMMddHHmmss }}` -> `1.37.*.{{ yyMMddHHmmss }}` -> `1.38.*.{{ yyMMddHHmmss }}`
    - 基线版本
    - 代号版本更迭
  </Card>


## 先锋渠道

> 先锋渠道是海文东标准的一部分

先锋渠道，包括实验性特性，由金丝雀渠道优化并裁剪开发测试套件等后传递，调试能力强，适合喜欢探索尝鲜的用户与贡献者。

  <Card title="Pioneer" icon="seti:plan">

        在 docker hub，先锋渠道使用 `Pioneer` 和 `P` 标签。

    更新周期较短（2周左右）

    - 发行名称：汐洛·先锋
    - 发行版本号更新 `0.100.*` -> `0.101.0` -> `0.101.1`
    - 内部版本号更新 `1.36.*.{{ yyMMddHHmmss }}` -> `1.37.1.{{ yyMMddHHmmss }}` -> `1.37.2.{{ yyMMddHHmmss }}`
    - 基线版本
    - 代号版本不变
  </Card>

> 金丝雀渠道是海文东标准的一部分

金丝雀渠道，包含了实验性特性，且包括开发测试套件等，具备完整调试能力，面向内部开发者。

    无更新周期。

    - 发行名称：汐洛·金丝雀
    - 发行版本号不变
    - 内部版本号更新 `1.36.1.{{ yyMMddHHmmss }}` -> `1.36.1.{{ yyMMddHHmmss }}`
    - 基线版本
    - 代号版本
  </Card>

## 新彖乄渠道

> 彖乄渠道是海文东标准的一部分

<Aside type="tip" title="注意">

彖乄渠道为特殊渠道，不对外发布，请勿上当受骗。

彖乄渠道标准的确立充分借鉴了汐洛安卓开发团队的实践经验，在此表示感谢。

原先计划的汐洛绞架单独发行呢？答：砍了。汐洛绞架只是汐洛矩阵中的普通组件，并没有能够独立发行的能量，对于有这方面需求的用户我们推荐使用思源笔记社区版。

</Aside>

全矩阵经过周期观察验证后才会推送新彖乄渠道更新，仅包含已冻结的特性，由洛可可渠道裁剪未冻结的特性后传递，解除所有限制。

<Card title="T" icon="seti:happenings">

在 docker hub ，新彖乄渠道使用 `T` 标签。

版本号更新与洛可可渠道一致。

     - 发行名称：汐洛
</Card>

彖乄渠道分为救赎（Salvation）与神罚（Damnation）两部分，救赎即矩阵应用的服务侧抽离出来独立发行，神罚即矩阵应用的应用侧抽离出来单独发行。
